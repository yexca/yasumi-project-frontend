# Standards

This directory stores the current repository conventions that contributors are expected to follow.

## Read These Files

- `code-organization.md`: placement and dependency rules.
- `documentation-maintenance.md`: how to keep `developer/` and `original/` clean over time.

## Questions This Directory Should Answer

- Where should new code live?
- What should stay out of shared components?
- How should synced versus local state be handled?
- When should documentation move to the archive?

## Maintenance Rules

- Keep standards concrete and enforceable.
- If a rule is really a one-off workflow, move it to `../workflows/`.
- If a rule is only true for one hotspot, move it to `../modules/`.
