# Phase Verification Archive

This directory preserves the original phase-by-phase implementation verification notes for the first development cycle.

## Reading Order

1. `phase-01-verification.md`: foundation and environment.
2. `phase-02-verification.md`: design tokens, themes, and primitives.
3. `phase-03-verification.md`: domain rules, DTO parsing, and repository contracts.
4. `phase-04-verification.md`: pages, routes, and shared item workflows.
5. `phase-05-verification.md`: local-first mutations and sync scaffolding.
6. `phase-06-verification.md`: i18n, settings, and local visual preferences.
7. `phase-07-verification.md`: testing, hardening, and release-readiness.

## When To Read

- You need to trace the order in which the current architecture emerged.
- You want historical acceptance boundaries for a subsystem.
- You are checking whether a current constraint started as a phase rule or is still a live standard.

## Archive Note

These records mention external guides and fixture workspaces that were used during the original implementation. They are preserved as historical references only.
