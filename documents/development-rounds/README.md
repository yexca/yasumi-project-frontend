# Development Rounds

This folder groups frontend documentation by delivery round.

## Rounds

- `round-1-original-development/`: archived original implementation records.
- `round-2-real-sync/`: active guidance for the real-sync and structure round.

## Maintenance Rule

Each round should be readable on its own. If a later round still needs guidance from an earlier round, merge the relevant content into the later round instead of asking implementers to jump across archived material.
